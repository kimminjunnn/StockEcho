# How It Works

StockEcho의 핵심 기능을 **어떻게 구현했는지** 설명하는 기술 문서를 모아 둔다.

이 폴더의 문서는 단순한 실행 방법보다 다음 내용을 중심으로 작성한다.

- 해결하려는 문제와 기존 방식의 한계
- 실제 데이터 처리 흐름
- 사용한 알고리즘과 모델
- 점수 계산과 판단 기준
- 구현 과정에서 선택한 구조와 이유
- 실제 입력과 출력 예시
- 현재 방식의 한계와 다음 개선 지점

## 문서 목록

### 뉴스 수집과 검색어 발견

- [뉴스 검색어를 자동으로 찾는 방법](./뉴스_검색어를_자동으로_찾는_방법.md): Kiwi와 TF-IDF, 제목 가중치, 언론사 다양성을 사용해 최근 뉴스에서 종목별 확장 검색어를 발견하는 방법
- [과거 유사 사례 뉴스 데이터는 어디서 가져올까](./과거_유사_사례_뉴스_데이터_소스_결정.md): BigKinds, NAVER, Google, Yahoo, GDELT, 크롤링과 OpenDART를 비교하고 NAVER 기반 MVP로 결정한 이유와 구현 범위

### 뉴스 Topic/Event 분석

- [주요 이슈 제목을 뽑는 방법](./주요_이슈_제목을_뽑는_방법.md): Kiwi 후보 구문, Event 임베딩, 기사 커버리지, MMR과 추출형 라벨링을 결합해 자연스러운 이슈명을 선택하는 방법

### 제품 기능

- [과거 유사 이슈를 찾고 주가를 비교하는 방법](./과거_유사_이슈를_찾고_주가를_비교하는_방법.md): 저장 Event 우선 검색, NAVER 보충, 핵심어 유사도, Supabase 캐시와 거래일 기준 수익률을 하나의 기능으로 연결한 방법
